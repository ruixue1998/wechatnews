我国日均词元（Token）调用量突破 140 万亿

People's Daily reported yesterday, according to the National Data Administration: 《人民日报》昨日报道，据国家数据局获悉：

In early 2024, China's average daily token call volume was 100 billion; by the end of 2025, it surged to 100 trillion; in March this year, it had exceeded 140 trillion, a thousandfold increase in two years. 2024 年初，中国日均词元（Token）调用量为 1000 亿；至 2025 年底，跃升至 100 万亿；今年 3 月，已突破 140 万亿，两年增长超千倍。

It is understood that tokens are the smallest information units processed by large models, possessing the characteristics of being measurable, priceable, and tradable in the intelligent era. 据了解，词元是大模型处理信息的最小信息单元，具有智能时代可计量、可定价、可交易的特征。

The report points out that the significant increase in China's average daily token call volume also indicates that with the deepening of reforms in the market-oriented allocation of data elements, a high-quality data supply system for artificial intelligence is taking shape, and a virtuous cycle of "data supply—value release" is beginning to emerge. 报道指出，我国日均词元调用量的大幅增长也表明，随着数据要素市场化配置改革的纵深推进，人工智能高质量数据的供给体系正在形成，「数据供给—价值释放」的良性循环初显。

Recently, APPSO also published an article mentioning that in academic literature on Natural Language Processing (NLP), Token, as the smallest processing unit of text, has long been translated as "词元" (cív yuán, token): 在近期，APPSO 也曾发文提到，在自然语言处理（NLP）学术文献里，Token 作为文本的最小处理单位，长期被译为「词元」：

“词” (cí, word) refers to linguistic attributes; “元” (yuán, element) refers to the smallest granularity. These two characters, one governs attribution, the other governs hierarchy. 词，指向语言属性；元，指向最小粒度。两个字，一个管归属，一个管层级。

According to Cailian Press, at the China Development High-Level Forum 2026 annual meeting recently, Liu Liehong, Director of the National Data Administration, stated that Token "词元" (cív yuán, token) is not only a value anchor in the intelligent era but also a "settlement unit" connecting technology supply and business demand, providing quantifiable possibilities for the implementation of business models; it also officially provided the translation for Token: "词元". 据财联社报道，日前在中国发展高层论坛 2026 年年会上，国家数据局局长刘烈宏表示，Token「词元」不仅是智能时代的价值锚点，更是连接技术供给与商业需求的「结算单位」，为商业模式的落地提供了可量化的可能；也从官方角度上为 Token 给出了翻译：「词元」。

Regarding another "old acquaintance" in the AI community—Prompt, APPSO also put forward its new translation suggestion: "文令" (wén lìng, text command). 而针对 AI 届另一个「老熟人」——Prompt，APPSO 也对翻译提出了自己新的建议：「文令」。

APPSO believes that compared to the currently well-known "提示词" (tí shì cí, prompt word), "文令" (wén lìng, text command) can better convey that the carrier of a Prompt is always text, and its function is to command and drive the model to execute. APPSO 认为，相较于目前大家熟知的「提示词」，「文令」能够表现出 Prompt 的载体永远是文字，以及 Prompt 的功能——下令驱动模型执行。

卢伟冰：手机越高端，内存涨价影响越小

According to Cailian Press, Lu Weibing, head of Xiaomi Group, stated at the earnings conference that the speed and intensity of this round of memory price increases are higher than the company originally imagined, posing a significant challenge to Xiaomi's operations. The lower the proportion of memory cost in the overall product, the smaller the impact on the product. 据财联社报道，小米集团负责人卢伟冰在业绩会上表示，这一轮内存上涨的速度、力度比公司原来想象得要高，对小米经营是一个很大挑战。内存成本占产品整体比例越低，对产品影响越小。

Lu Weibing also stated that Xiaomi's "human-car-home full ecosystem" moving towards balance will allow the company to better cope with memory price increases. At the same time, the more high-end the phone, the smaller the impact. 微信插件被 OpenClaw 更新「干崩」

Recently, OpenClaw released version 2026.3.22-beta.1, undergoing a major refactoring involving the plugin system, which subsequently led to the official WeChat ClawBot plugin becoming inoperable after the OpenClaw update. 近日，OpenClaw 发布 2026.3.22-beta.1 版本，进行了一次涉及插件系统的重大重构，进而导致微信官方 ClawBot 插件在 OpenClaw 更新后陷入无法运行的状态。

It is reported that the core change in this refactoring was the removal of the original unified module entry openclaw/plugin-sdk, forcing all plugins to use subdivided paths (e.g., openclaw/plugin-sdk/core) to load required functions on demand. 据悉，此次重构的核心改动，是删除了原有的统一模块入口 openclaw/plugin-sdk，强制要求所有插件改用细分路径（如 openclaw/plugin-sdk/core）按需加载所需功能。

According to OpenClaw's official explanation, this refactoring aims to achieve on-demand loading to improve startup speed and reduce memory usage, while also strengthening security isolation by blocking cross-package access via relative paths, preventing plugins from unauthorized reading of other local data. 据 OpenClaw 官方解释，此次重构旨在实现按需加载，以提升启动速度、降低内存占用，同时通过阻断相对路径的跨包访问来强化安全隔离，防止插件越权读取本地其他数据。

However, the update log explicitly states 'no compatibility shim', meaning no compatibility transition plan is provided. Because the WeChat Clawbot plugin code relies on the original path, OpenClaw could not find the corresponding module in the runtime environment, ultimately rendering all conversations with Clawbot within WeChat completely invalid. 但更新日志中明确注明「no compatibility shim」，不提供任何兼容过渡方案。由于微信 Clawbot 插件代码依赖原有路径，导致 OpenClaw 在运行环境下找不到对应模块，最终微信内与 Clawbot 的所有对话完全失效。

Related plugins for multiple IM channels such as WeChat Work and Feishu were also affected during the same period; although QQ Bot triggered static code scanning warnings during the update process, actual use was not affected because the warnings did not block operation. 同期受波及的还包括企业微信、飞书等多个 IM 渠道的相关插件；QQ Bot 虽在更新过程中触发了静态代码扫描警告，但由于警告不阻断运行，实际使用未受影响。

After the incident escalated, WeChat employee '@Kecun Xiao Jiang' responded on Weibo yesterday, stating, 'We will update the plugin soon to fix this issue,' and added, 'This upgrade of OpenClaw seems to have also affected a bunch of IM message channels both domestically and internationally. It's understandable to have minor issues in a new product iteration.' 事件发酵后，微信员工「@客村小蒋」昨日在微博回应称「我们很快会更新下插件，解决这个问题」，并表示「openclaw 的这次升级，似乎也影响了一堆国内外的 IM 消息渠道，一个新产品的迭代里有点小问题，可以理解」。

According to Tencent PR Director Zhang Jun's message yesterday afternoon, the WeChat Clawbot plugin has now been updated and related issues fixed. 据腾讯腾讯公关总监张军昨日下午消息，目前微信 Clawbot 插件已完成更新并修复相关问题。

According to media reports, this OpenClaw update also caused the browser extension's Relay function to become invalid due to path removal, abnormal configurations for domestic models like MiniMax, and permission errors in Windows Sandbox. 另据媒体报道，OpenClaw 本次更新还导致浏览器扩展 Relay 功能因路径移除而失效，MiniMax 等国产模型配置异常，Windows 沙箱出现权限错误。

'Father of Lobster' Peter Steinberger also posted that a step for handling 'web control interface resources' was manually missed when releasing the new version. To avoid repeating such 'manually missed steps' errors in the future, he is currently automating the entire software release process and adding end-to-end testing for the web client; this way, after the code is written, the machine will automatically perform comprehensive tests and package the release. 为了避免以后再犯这种「人为漏掉步骤」的失误，他目前正在把整个软件的发布流程做成自动化，并且正在为网页端添加端到端测试；这样以后代码写完后，机器会自动进行全面测试并打包发布。
